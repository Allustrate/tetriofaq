术语

和其它社区一样，TETR.IO 社区使用众多新人不为所知的“行话”交流。本章节将给出一张列表，列举一些可能让玩家疑惑的术语。参见：

  • Tenchi 的缩略词词典；
  • 俄罗斯方块中文维基的术语表 - 搬运自 Hard Drop Wiki 与 Tetris Wiki，有些过时；
  • 俄罗斯方块中文维基上 TETR.IO 条目的琐事章节；
  • four.lol - 可直接输入术语缩写，如tki、mko、tst、dpc、甚至 sd-pc；
  • TETR.IO 游戏本身的悬浮文本也可当做“术语表”，尤其是 CONFIG 与 TETRA CHANNEL。

符号系统

游戏的用户界面有时会使用一些符号表达信息，下面给出其中部分符号的解释。

禅意模式等级zenlevels

符号与阿拉伯数字的关系示意图。

TETR.IO 中，禅意模式的等级旁边通常会带有一串符号。这些符号以类罗马数字的形式表示了禅意模式的等级。

达到最高等级时，该系统会重新开始计数。但截至目前，尚未有玩家完成这一循环，目前禅意模式等级最高的玩家离循环还差 490,605 级。

单人模式选项solo-options

在开始40行与闪电战之前，玩家可调整一些额外的选项以自定义游戏体验。这些选项主要影响游戏的画面，对于专业玩家，这些选项没有默认的推荐配置。 有关极简操作的内容不在本节内详述，可在中文维基或 Tehchi 的 FAQ 中找到相关内容。

查看游戏内的悬浮文本可解决大部分疑惑，请善用。

PRO MODEpromode

该选项较为显眼，且在40行中实用性高。开启该模式可在40行中依次额外显示以下信息：

  1. 场地背景显示剩余行数或剩余时间；
  2. 总按键数与每块按键数；
  3. 极简连击、极简率与极简失误数。

PRO MODE 在早期就加入了游戏，但玩家可在下方的选项中复制 PRO MODE 的配置。例如，如果觉得场地中的剩余行数会干扰视线，但想要保留其他有用的信息，可按以下方式配置：

  • Left counter slot 1: STOPWATCH
  • Left counter slot 2: LINES
  • Left counter slot 3: PIECES
  • Left counter slot 4: KEYS
  • Right counter slot: FINESSE

STRIDE MODEstridemode

开启该选项后，开局、结束与重开的动画将缩短。其中最显著的改变是“按键重开”，旨在减少玩家在重开上耗费的时间。此外，该选项还会影响如下如下内容：

  • 方块生成：首块不会是 S、Z、O 三者之一
  • 认输耗时更短
  • 开局倒计时的速度变快

社区黑话

TETR.IO 社区创造了各种不同形式的术语来描述战术和玩法，其中有一些只适用于无延迟方块。下面举几个例子。

"Plonker"plonk

"Plonking" 指在游戏中有有意接收大量垃圾行，并以此利用垃圾行打出大量反击伤害的行为，有时玩家会在收垃圾行时明显降速。 这种利用垃圾行的战术适用于 TETR.IO 中“随攻击改变孔洞位置”的垃圾行设置。这种设置下，每次攻击垃圾行的孔洞位于同一列，有随机器决定。 "plonker" 可用于泛指游戏中 VS:PPS 比值较高的玩家（VS 很大程度上受挖掘垃圾行的数量影响，因此比例的前项是 VS 而不是 APM）。 实战中，"plonker" 是 "strider" 的反义词："plonker" 通常速度较慢，注重垃圾行，且场地高度较低。

"Strider"strider

该梗源自 Jstris 社区，常用来称呼符合以下条件的玩家：极快的40行成绩（低于26.179s）、“不会变成悲伤蛙”、与40行成绩严重不符的对战水平、在实战与40行中飚速度、个人资料使用 🇮🇲 区域旗帜。strider 通常指 PPS:APM 比值较高的玩家，也即高速低效的玩家。通常，只有无延迟方块会使用 strider 这个词。 "stride" 与 "strider" 可互换，但 "striding" 指在单局游戏中突然提速降效。
