TETRA LEAGUE

TETRA LEAGUE（四联联盟、TL）是 TETR.IO 的排位模式，供所有玩家免费游玩，只需注册账户并达到 10 级即可参与。联盟的赛制十分简单——与相近技能水平的玩家展开单挑对战，但部分玩家可能会好奇该模式底层的工作原理与技术参数，本小节将就一些常见疑问展开解释。

评分偏差值rd

评分偏差值是 Glicko-2 的一次扩展，其中 Glicko-2 正是 TETRA LEAGUE 使用的评分系统。Glicko 起初用于国际象棋比赛，后被众多游戏与组织用作评分系统。

RD 是游戏中 Glicko 评分中±之后的数值，其全程是Rating Deviation，也即偏差值。偏差值体现了“你 Glicko-2 评分不确定的程度，越低越好。”（游戏内文本）

一场排位赛后，偏差值通常会降低，但也有增高的情况。据 osk，偏差值不会低于60，但也有一些玩家达到过58的偏差值。偏差值在达到 349 之后就不会继续增长。

偏差值会在连续七天未进行排位比赛后，以每天一点的固定速率增加。可在四连频道个人用户页 https://ch.tetr.io/u/[username] 中查看你是否处于“不活跃状态”。若你的 Glicko 段位分旁边有 Ƿ 的符号，则说明你有一周没打排位了。

操作参数

提示： 您可随时通过 HANDLING 小节右上角的 TEST 按钮测试当前的操作参数。

操作参数控制玩家持续按键时，方块的移动属性。玩家可在 CONFIG 设置选单中 HANDLING 部分调解它们。下面将简要介绍这些参数。

参见：

  • YouTube：25Pi25's：How to properly set your TETR.IO Handling!（英语，关于如何正确设置 TETR.IO 操作参数）
  • 一张直白介绍 DAS 与 ARR 的 .gif 动图 （Zaptor：所有人都在分享这张图片，但我一直找不到原作者。）
  • Tenchi 的 FAQ： Jstris 与 TETR.IO 在操作参数上的差异。
    • 简而言之，这两个平台的 DAS 与 ARR 满足以下换算公式：

Auto Repeat Ratearr

[A]uto [R]epeat [R]ate（移动重复延迟，越低越快，零为瞬移）：ARR 影响方块在持续按下向左或向右时，方块移动的速率。当 ARR 为零时，方块在 DAS 充能完毕后就会撞墙，这样极大提升了操作速度，因此有人将零 ARR 称为“最优” ARR。然而，其性能几乎只有在极简操作下才能得到完全发挥

Delayed Auto Shiftdas

[D]elayed [A]uto [S]hift（自动移动延迟，越低越快）：ARR 生效前需要持续按下方向键的时间。简而言之：

    Let the bass kick O-----o-o-o-o-o-o-o-o-o-o
  • O 和第一个 o 的等待时间可理解为 DAS。
  • 后续 o 之间的重复时间可理解为 ARR。

“最优” DAS 因人而异，所以在实战前请多多尝试不同的参数。

DAS Cut Delaydcd

[D]AS [C]ut [D]elay（DAS 打断延迟，越高越慢，若为零则不生效）：作为一项较复杂的实验性参数，高端玩家很少特意使用 DCD。DAS 打断将在以下情况下生效，打断期间内，DAS 无法充能：

方块旋转

新方块生成

进行上述两种操作时，DAS 停止充能一段时间，这段时间即为 DCD。DCD 主要针对 0ARR 环境，旨在让 DAS 充能的情况下完成特定的极简操作。该参数也可以避免操作失误。

Soft Drop Factorsdf

其实应该叫软降速率倍率的……但是太拗口了

[S]oft [D]rop [F]actor（软降倍率，越搞越快：∞ 是瞬降）：该参数控制方块在按下软降时，方块降落的速度。该参数并不一定要乘上当前的重力，毕竟零重力下方块还是可以软降。（小学数学：零乘任何数都是零，即使是无穷大）

推荐将这一参数设置为∞，毕竟需要大量软降的堆叠只能算是不太完美的半完整堆叠。

如果你只能适应 20X 以下的软降倍率，那就先好好回去练堆叠吧。

瞬降sonicdrop

想要完成“不锁定方块的硬降”，比如快速 T-Spin，就把 SDF 拉到∞吧！这样，一按下软降，方块就会立马接地！

S-Spin 与 Z-Spinzspins

左转、左转
右转、右转

右转、右转
左转、左转

由于 SRS 在诞生时，设计者并没有考虑太多，只是给“对称”下了几个反常的定义，新玩家可能较难理解 S-Spin 与 Z-Spin，。正是这些定义让他们常常无法完成 S-Spin 与 Z-Spin。

如果没有墙，完成 Z-Spin 需要从初始朝向逆时针旋转一次，软降，然后再用逆时针把 Z 块塞进去。反之，若两边都有墙（如 gif 的后半部分），就需要顺时针旋转两次了。

Tip: 若使用默认键位，那么 Z-Spin 就需要按两次 "z" 键。

如果没有墙，完成 S-Spin 需要从初始朝向顺时针旋转一次，软降，然后再用顺时针把 S 块塞进去。 （大多数新手只会使用上键顺时针单项旋转，所以 S-Spin 对新玩家可能“更简单”。）
如果两边都有墙（如 gif 的后半部分），就需要调换旋转方向，逆时针旋转两次。

/set 指令用法

游戏中 /set 指令有多重使用场合，大部分由社区玩家开发。了其中主要指令的用法，这样你就不用在游戏中花时间打一大串指令了。

自动填充setgen

TETR.IO 为自定义房间的房主提供了若干预设。房主只需前往 GAME 选项卡中的第一个选项，在六个预设中选择其一即可。玩家现无法创建自定义的预设，但这项功能可能会在将来加入游戏。

首先将介绍若干自动填充的方法，按对游戏底层代码的影响程度排列，从低到高：

  • craftxbox 的 Continued Autohost 项目 有若干有关房间预设的指令。详请参考该 游戏内 bot 的 说明文档。将你的房间设置完毕后，使用指令!savepreset <name> 来将其保存为一套预设。之后就可通过 !preset <name> 来加载该预设，或是通过 !delpreset <name> 来删除它。该 bot 同时自带一些预设，可通过 !preset 浏览。
  • Zutatensuppe的自定义 JavaScript 书签：英文教程及书签的源代码请见 该链接 and 该链接。
  • aznguy.mp4 的 控制台解决方案：将该 Github Gist 复制粘贴到浏览器的开发者工具（大多通过 F12 打开，包括 TETR.IO 桌面版），以将当前房间的设置导入控制台。该方法对游戏底层代码影响较大，且需要输入大量额外默认的定义，因此只适用于从零新建 /set 预设的情况。

手动修改manualset

如果你想自己动手，丰衣足食，以下给出一些方法：

  • aznguy.mp4给出了一张 /set 属性的列表。该列表同时挂在 TETR.IO 官方 Discord 服务器中 #tetrio 频道的置顶消息，你也可以通过 Tenchi 的 FAQ找到这张列表。
  • ZaptorZap写了五种模拟不同环境的 /set 属性，可通过该链接下载（若有新的建议，欢迎通过 ZaptorZap#0405 向其提出）。

注意：TETR.IO 最多允许用户在聊天框输入512个字符，/set 指令也不例外。若指令过长，可将其拆成两半，或是删除默认指令。

概括性机制

以下是一些较难分类的机制。玩家经常会误解这些机制，甚至压根不了解它们。

火力全开fire

达到足够的“火力点”后，即可开启“火力全开”特效。瞬发攻击 Spike、背靠背连锁、击败他人都可积攒火力点，其中击败他人获得的火力点最多。击败他人将获得他积攒的火力点，KO 旁的百分数即通过击败获得的火力点数。整个“火力全开”系统只是游戏特效，并可在设置中关闭。
